# Quickstart

This guide shows you how to create a payment order and redirect a customer to the payment page.

## Step 1: Get your API key

Contact your C2C admin to generate a merchant API key for your account.
Your key will look like: `mk_live_a1b2c3d4e5f6...`

<Warning>
  Store your API key securely. Never expose it in client-side code or commit it to version control.
</Warning>

## Step 2: Create an order

From your backend server, send a `POST` request to create an order.

<CodeGroup>

```bash cURL
curl -X POST https://ca-c2c-dev-api.calmsea-c64629ed.uaenorth.azurecontainerapps.io/api/merchant/orders \
  -H "Authorization: Bearer mk_live_your_api_key" \
  -H "Idempotency-Key: order_$(uuidgen)" \
  -H "Content-Type: application/json" \
  -d '{
    "orderType": "Deposit",
    "amount": 1500.00,
    "currency": "PKR",
    "walletType": "JazzCash",
    "merchantOrderRef": "ORD-20260618-7890",
    "customerReference": "player_12345"
  }'
```

```javascript Node.js
const response = await fetch('https://ca-c2c-dev-api.calmsea-c64629ed.uaenorth.azurecontainerapps.io/api/merchant/orders', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.C2C_API_KEY}`,
    'Idempotency-Key': crypto.randomUUID(),
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    orderType: 'Deposit',
    amount: 1500.00,
    currency: 'PKR',
    walletType: 'JazzCash',       // required
    merchantOrderRef: 'ORD-20260618-7890', // optional: your own reference
    customerReference: 'player_12345',
  }),
});
const order = await response.json();
console.log(order.data.paymentPageUrl);
```

```python Python
import httpx, os, uuid

response = httpx.post(
    'https://ca-c2c-dev-api.calmsea-c64629ed.uaenorth.azurecontainerapps.io/api/merchant/orders',
    headers={
        'Authorization': f'Bearer {os.environ["C2C_API_KEY"]}',

        'currency': 'PKR',
        'walletType': 'JazzCash',          # required
        'merchantOrderRef': 'ORD-20260618-7890',  # optional
        'customerReference': 'player_12345',
    },
)
order = response.json()
print(order['data']['paymentPageUrl'])
```

## Step 3: Redirect the customer

The response includes a `paymentPageUrl`. Redirect your customer to it immediately:

```json
{
  "success": true,
  "data": {
    "orderId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "orderReference": "C2C-20260618-001234",
    "merchantOrderRef": "ORD-20260618-7890",
    "paymentPageUrl": "https://payment.c2cplatform.com/pay?order_id=a1b2c3...&signature=xyz&expires=...",
    "status": "Pending"
  }
}
```

The payment page automatically shows the assigned member's account number and wallet type. Your customer does not need to select an account — the platform picks the best available member based on online status.

<Warning>
  The payment page URL expires after **15 minutes**. Redirect the customer immediately — do not cache or reuse these URLs.
</Warning>

On the payment page, the customer:
1. Sends money to the displayed account number
2. Enters the **last 4 digits** of their transaction ID
3. Optionally uploads a receipt image
4. Submits — the assigned member is then notified to verify

## Step 4: Poll for confirmation

After the customer submits, the assigned member verifies the payment and approves or rejects it.
Poll `GET /api/merchant/orders/{id}/status` every 3–5 seconds until resolved:

```javascript
const terminalStatuses = new Set(['Completed', 'Confirmed', 'Rejected', 'Cancelled', 'CancelledAuto', 'Expired', 'ManualReview']);

async function pollUntilDone(orderId) {
  while (true) {
    const { data } = await getOrderStatus(orderId);
    if (terminalStatuses.has(data.status)) return data.status;
    await new Promise(r => setTimeout(r, 5000));
  }
}

const finalStatus = await pollUntilDone(order.data.orderId);
if (finalStatus === 'Completed' || finalStatus === 'Confirmed') {
  // Credit coins or fulfil the purchase
}
```

<Note>
  Set `callbackUrl` in your order creation request to receive a webhook instead of polling.
  See the [Webhooks guide](/webhooks) for details.
</Note>

## Withdrawal orders

For withdrawal orders (where a member sends money **to** your customer), pass `orderType: "Withdrawal"` and include the customer's account details:

```bash cURL
curl -X POST https://ca-c2c-dev-api.calmsea-c64629ed.uaenorth.azurecontainerapps.io/api/merchant/orders \
  -H "Authorization: Bearer mk_live_your_api_key" \
  -H "Idempotency-Key: withdraw_$(uuidgen)" \
  -H "Content-Type: application/json" \
  -d '{
    "orderType": "Withdrawal",
    "amount": 2000.00,
    "currency": "PKR",
    "walletType": "Easypaisa",
    "merchantOrderRef": "WITHDRAW-20260618-0042",
    "customerAccountNumber": "03211234567",
    "customerReference": "player_12345"
  }'
```

The assigned member sees the customer's account number on their app, sends the money, confirms with the last 4 digits of their outbound transaction, and the order moves to `Confirmed`.
