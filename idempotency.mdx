# Idempotency

The `POST /api/merchant/orders` endpoint requires an `Idempotency-Key` header.
This lets you safely retry failed requests without creating duplicate orders.

## How it works

When you send a request with an `Idempotency-Key`:

1. If the key is **new**, the order is created and the response is stored
2. If the key is **already used with the same parameters**, the original response is returned — no new order is created
3. If the key is **already used with different parameters**, a `409 Conflict` is returned

## Sending the header

```http
POST /api/merchant/orders HTTP/1.1
Authorization: Bearer mk_live_your_api_key
Idempotency-Key: order_player_12345_20260519_001
Content-Type: application/json
```

## Key format recommendations

Use a key that uniquely identifies your intent — not just a random UUID:

```javascript
// Good: combines your order reference and customer
const idempotencyKey = `order_${customerId}_${yourInternalOrderId}`;

// Also fine: a UUID generated once per checkout attempt
const idempotencyKey = crypto.randomUUID();
```

<Warning>
  Generate the key **once** per order attempt and persist it in your database
  before sending the request. If you generate a new key on each retry, you will
  create duplicate orders.
</Warning>

## Idempotency window

Keys are stored for **24 hours**. After that, the same key can be reused for a new order.

## Handling conflicts

A `409 Conflict` means the key was already used with different request parameters:

```json
{
  "success": false,
  "error": "Idempotency key already used with different parameters.",
  "code": "IDEMPOTENCY_CONFLICT"
}
```

This is not an error in the usual sense — it is a safety mechanism.
Check your original order in the dashboard and use a new key for the retry.

## Safe retry pattern

```javascript
async function createOrderWithRetry(params) {
  // Generate the idempotency key ONCE and store it before the first attempt
  const idempotencyKey = `order_${params.customerId}_${params.internalOrderId}`;

  const response = await fetch('https://ca-c2c-dev-api.calmsea-c64629ed.uaenorth.azurecontainerapps.io/api/merchant/orders', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.C2C_API_KEY}`,
      'Idempotency-Key': idempotencyKey,  // same key on every retry
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(params),
  });

  // 409 = already processed — return the existing order safely
  if (response.status === 409 || response.ok) {
    return response.json();
  }

  const error = await response.json();
  throw new Error(error.error);
}
```
